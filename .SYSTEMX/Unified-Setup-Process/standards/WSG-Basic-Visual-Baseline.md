# WSG Basic Visual Baseline

The default generated app starts intentionally plain so each project can add its
own brand later.

## Color Rule

Use only black, white, and neutral gray in the base template.

| Token | Example | Use |
| --- | --- | --- |
| `white` | `#ffffff` | Page background |
| `neutral-50` | very light gray | Footer / subtle bands |
| `neutral-200` | light gray | Borders |
| `neutral-500` / `neutral-600` | mid gray | Supporting text |
| `neutral-950` | near black | Headings, primary buttons |

Avoid brand colors, gradients, decorative color accents, and theme-specific
palettes in the base starter. Project-specific editions may add a brand system
after setup.

## Starter Pages

The validation starter includes:

- Home
- About
- Services
- Docs
- Contact
- 404 fallback

Every visible page should read as a generic template until a project purpose is
selected.

## Navigation

The standard menu is a hamburger menu on every viewport. Do not show a separate
desktop horizontal nav in the base template. Project-specific apps may replace
this after selecting an edition and brand system.
