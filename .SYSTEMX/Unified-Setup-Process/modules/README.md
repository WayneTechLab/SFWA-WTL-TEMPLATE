# Module Packs

Modules are reusable setup capabilities selected by edition manifests.

| Module | Purpose |
| --- | --- |
| auth | Authentication, route guards, role placeholders |
| commerce | Stripe/client checkout/server webhook placeholders |
| firebase | Firebase Hosting, Firestore, Storage, app config |
| functions | Cloud Functions and server-only secrets |
| ci | GitHub Actions and npm script gates |
| mcp | MCP/dev browser validation loops |
| security | Rules, secret hygiene, MFA/App Check/RBAC guidance |
| monitoring | Sentry/GCP/Firebase monitoring and alerts |
| pages | Page registry and route/page budget |
| docs | README/wiki/operator docs |
| runbooks | Incident, rollback, deploy, and secrets runbooks |
| repo-learning | Authenticated repo manifest scrape and reusable pattern extraction |
| dev-tooling | Local dev, test forks, WSGT/WSGD tooling |

Module docs describe placeholders and standards only. Do not copy
project-specific business logic into the reusable template.
