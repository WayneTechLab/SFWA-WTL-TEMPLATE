# 10 Content And SEO Standard

Use this file when an LLM creates or updates page copy, metadata, sitemap
strategy, content models, docs, or marketing pages.

## Content Rules

- Keep template copy generic until a project brief defines a brand.
- Write in direct, useful language.
- Do not invent claims, testimonials, certifications, pricing, team members, or
  legal promises.
- Separate durable template language from project-specific business language.
- Prefer short headings that describe the actual page or action.
- Do not use visible in-app text to explain implementation details or keyboard
  shortcuts unless the user asked for a help/training surface.

## Required SEO Fields

Each public project should define:

- Site name.
- Default title.
- Default description.
- Canonical domain.
- Robots policy.
- Sitemap route list.
- Open Graph title.
- Open Graph description.
- Open Graph image.
- Structured data needs.

## Route Content Checklist

For each public route, define:

- Route path.
- Page title.
- Meta description.
- Primary H1.
- Audience.
- Search intent.
- Account level.
- Index/noindex decision.
- Required media.
- Conversion/action target.

## Template SEO Default

The starter may say:

```text
Web Stack Generation - Template Provided by Wayne Tech Lab LLC.
```

Replace this during project setup with the completed project brief.
